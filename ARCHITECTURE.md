## Core Data Flow

The complete lifecycle from element selection to file persistence:

```mermaid
sequenceDiagram
    participant User
    participant Overlay
    participant Infer as infer.ts
    participant Panel as WebflowPanel
    participant Apply as apply.ts
    participant LS as localStorage
    participant Server as server/commit.ts
    participant FS as Source Files

    User->>Overlay: Press backtick (`) — enter selection mode
    User->>Overlay: Click element
    Overlay->>Infer: infer(element)
    Note over Infer: getComputedStyle()<br/>→ detect type (text? flex? grid?)<br/>→ build DialConfig sections
    Infer-->>Panel: InferResult { config, name, varUnits, spacing }
    Panel->>Panel: Render 8 collapsible sections + Custom properties

    User->>Panel: Drag slider / pick color
    Panel->>Apply: applyInlineStyle(el, prop, value)
    Note over Apply: 1. Record initial value<br/>2. Push undo stack<br/>3. el.style.setProperty(prop, val, "!important")<br/>4. Notify listeners
    Apply->>LS: schedulePersist() — async serialize

    User->>Panel: Click "Save"
    Panel->>Apply: diff(element) → DiffEntry[]
    Panel->>Server: POST /api/tuner/commit { changes, mode }
    Note over Server: 1. Resolve source file (sourcemap → path → search)<br/>2. Find CSS property in file<br/>3. Surgical string replacement
    Server->>FS: Write modified file
    FS-->>Overlay: HMR hot reload → styles update
```

## File Persistence Pipeline

The server-side commit process uses a tiered resolution strategy:

```mermaid
graph TD
    Start["Receive DiffEntry[]<br/>{ prop, from, to, sourceFile?, sourceLine? }"]

    Start --> Resolve["Resolve Source File"]

    Resolve --> T1{"Has sourcemap<br/>href?"}
    T1 -->|Yes| SM["Read .map file<br/>→ resolve original path"]
    T1 -->|No| T2{"Has direct<br/>path hint?"}
    T2 -->|Yes| DP["Validate path exists"]
    T2 -->|No| T3["Recursive filesystem search<br/>(exclude node_modules, .next, dist)"]

    SM --> Found
    DP --> Found
    T3 --> Found

    Found["Source file located"]
    Found --> Find["Find CSS property in file"]

    Find --> S1{"±5 line<br/>window?"}
    S1 -->|Yes| Replace
    S1 -->|No| S2{"Class block<br/>search?"}
    S2 -->|Yes| Replace
    S2 -->|No| S3{"Full file<br/>scan?"}
    S3 -->|Yes| Replace
    S3 -->|No| S4["Fuzzy match"]
    S4 --> Replace

    Replace["Surgical string replacement<br/>e.g. color: red → color: blue"]
    Replace --> Write["fs.writeFile()"]
    Write --> HMR["HMR detects change<br/>→ hot reload"]
```

## Undo/Redo System

```mermaid
stateDiagram-v2
    [*] --> Clean: Panel opens

    Clean --> Modified: applyInlineStyle()
    Modified --> Modified: More changes (coalesce same-prop)

    Modified --> Undone: Ctrl+Z / Cmd+Z
    Undone --> Modified: Ctrl+Shift+Z (redo)
    Undone --> Undone: More undos
    Undone --> Clean: Undo all

    Modified --> Batch: startBatch()
    Batch --> Modified: endBatch() → single undo entry

    Modified --> Saved: Click "Save" → commit to files
    Modified --> Clean: Click "Reset" → restore all initials
```

**Key behaviors:**
- **Coalescing** — consecutive drags on the same property don't create multiple undo entries
- **Batching** — multi-step operations (e.g., paste styles) collapse into one undo entry
- **State-keyed** — pseudo-class overrides (`:hover`, `:focus`) stored separately with `"state::prop"` composite keys
- **Max depth** — 200 entries, oldest dropped on overflow

## Hotkey Reference

All shortcuts live in one capture-phase listener (`hooks/useOverlayHotkeys.ts`);
the in-app `?` overlay (`shell/ShortcutsHelp.tsx`) is the canonical list.

| Key | Action |
|-----|--------|
| `` ` `` (backtick) | Toggle selection mode / close panel |
| `Escape` | Dismiss modal → close search → close panel |
| `Cmd/Ctrl + Z` | Undo |
| `Cmd/Ctrl + Shift + Z` | Redo |
| `Cmd/Ctrl + S` | Save to source |
| `Cmd/Ctrl + C` | Copy CSS |
| `Cmd/Ctrl + Alt + C` / `+ V` | Copy / paste styles between elements |
| `Cmd/Ctrl + Shift + V` | Import CSS from clipboard |
| `Cmd/Ctrl + K` | Command palette |
| `Cmd/Ctrl + F`, `/` | Property search |
| `D` (hold) | Diff peek (strips overrides while held) |
| `S` | Cycle scope (element ↔ class) |
| `Alt + Shift + S` | Toggle focus mode |
| `R` / `Shift + R` | Reset element / reset all |
| `P` | Pin / unpin element |
| `N` | Toggle navigator |
| `H` | Toggle changes drawer (history tab) |
| `M` / `G` | Toggle box model / grid overlay |
| `T` | Toggle Style / AI tab |
| `1`–`8` | Jump to section |
| `[` / `]` | Cycle sections |
| `↑ ↓ ← →` | Navigate elements (parent / child / siblings) |
| `?` | Shortcuts help |

**Context-aware pass-through** (`hotkeyPageHijacking.test.tsx` pins this matrix —
the overlay must never hijack the host page):

- `Cmd+F` / `Cmd+K` — claimed **only** while focus is inside the tuner UI
  (`.__tuner-root` or `[data-tuner-portal]`). On the page, browser find and
  host command palettes keep working.
- `Cmd+C` — passes through when the page has a text selection, when a text
  control holds a range selection, or when focus is in a host editing surface;
  otherwise claimed (copy CSS).
- `Cmd+Z` / `Cmd+Shift+Z` — claimed inside the tuner UI (including panel
  inputs); pass through in host text fields (native undo wins); on the page,
  claimed only when the style engine actually has a step to revert/replay.
- `Cmd+S` — claimed globally while the panel is open (blocks the browser save
  dialog).
- Plain-key shortcuts (`S`, `R`, `D`, arrows, digits, …) never fire while
  typing in inputs or while focus is inside the panel/portals.
