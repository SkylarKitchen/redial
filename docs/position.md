# Why Redial Exists

Webflow proved that visual CSS tuning belongs in a GUI, not a text editor. Drag a slider, see the result, move on. But Webflow is a platform — it owns your hosting, CMS, and deployment. If you build in Next.js, you're back to editing files and waiting for HMR. A change that takes two seconds with a slider takes thirty in a text editor, across hundreds of micro-decisions that separate polished UI from merely functional.

The tools that exist don't close the gap. DevTools changes vanish on reload. AI coding tools add minutes of latency per visual tweak — you describe intent in words, wait for inference, check the result, and iterate. Direct manipulation would resolve it instantly.

Redial is the missing layer. Click an element, get context-aware controls inferred from its computed styles, drag to tune with zero latency, and save directly to your source files through HMR. No copy-pasting. No prompt engineering. No platform lock-in. Your code stays your code — Redial just gives you a faster way to shape it.
